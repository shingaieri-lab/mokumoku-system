メール配信手段の選択肢（一番大きな分岐）

数千件の一斉配信は既存の Gmail API では不可能。専用の配信サービス（メール配信プロバイダー）を導入する必要があります。

リスク

懸念点・リスク整理

実装着手前に必ず方針を決めておきたい7つのリスク。重要度順。
🔴 高

1. リストの入手元・オプトインの法的扱い

何が問題か：特定電子メール法では、原則「事前に同意を得た人」にしか広告メールを送れません（オプトイン規制）。名刺交換・問い合わせ等の例外もありますが、購入リスト・スクレイピングリストは違法の可能性が高いです。

確認したい：送信するリストはどこで入手したものか？同意取得の根拠は何か？

対策案：リストの入手経路（名刺交換・展示会・問い合わせ等）を CSV インポート時に必須入力にする。同意根拠が無いリストは配信できない仕様にする。

🔴 高

2. スパム判定によるドメイン全体への悪影響

何が問題か：大量配信でスパム報告が一定数を超えると、送信ドメイン全体の評判が落ち、通常の業務メール（取引先・顧客連絡）まで届かなくなります。

対策案：キャンペーン専用のサブドメイン（例：news.dandori-work.com）を新規取得して切り離す。本社メインのドメインを守る。配信ペースもいきなり数千件ではなく徐々に増やす（ウォームアップ）。

🔴 高

3. 「興味あり」応答者の対応遅延リスク

何が問題か：「興味あり」と回答した人を放置すると、印象が逆に悪化します（「興味あると押したのに連絡来ない」というクレームに直結）。

対策案：架電リスト追加時に担当ISへメール／Slack通知。応答から24時間以内の架電をSLAとして設定。ダッシュボードで未対応件数を可視化。

🟡 中

4. 数千件CSVインポート時の処理時間

何が問題か：Vercel Function のタイムアウトは現在300秒。数千件のCSVをパース＋重複チェック＋KV保存をシリアル処理すると、件数次第で詰まる可能性。

対策案：バッチ処理（100件ずつ非同期で保存）。インポート画面に進捗バー表示。10,000件超なら分割アップロードを推奨。

🟡 中

5. 配信途中での失敗・重複送信

何が問題か：3,000件配信中に1,000件目でエラー停止した場合、どこから再送するか管理が必要。重複送信は信頼失墜に直結。

対策案：recipient ごとに sent_at を都度記録。再送ボタンは sent_at が null のものだけを対象。冪等（べきとう）性を担保する。

🟢 低

6. Vercel KV のデータ容量

何が問題か：キャンペーン×数千件×蓄積で、半年で数万レコードに膨らむ可能性。

対策案：recipient データは6か月で自動アーカイブ（または手動エクスポート後削除）。応答者は lead 側に保存済みなので recipient 側を消してOK。

🟢 低

7. フォーム応答の不正回答（連打・なりすまし）

何が問題か：トークンURLが第三者に流出すると、なりすまし回答が可能。「興味あり」のいたずら回答もあり得る。

対策案：応答は1回まで（上書き不可）。IPアドレス単位のレート制限。トークンは推測不可能な十分な長さ（24文字以上のランダム文字列）。

確認チェックリスト（実装着手前に決めること）

この17項目が固まれば、Phase 1 から着手できます。

🔧 技術選定（3項目）

  • □ 配信プロバイダー：Resend / SendGrid / Amazon SES のどれを使うか
  • □ 送信元ドメイン：既存ドメインのサブドメイン or 専用ドメイン新規取得
  • □ DKIM/SPF/DMARC 設定：誰がいつ実施するか（DNS設定変更が必要）

📊 規模感（3項目）

  • □ 月間配信回数：月1回？週1回？
  • □ 1回あたりの配信件数：数千件想定でOK？それ以上はあり得るか
  • □ 月額予算上限（配信サービス料＋ドメイン代）

⚖️ コンプライアンス（3項目）

  • □ 配信リストの入手経路と同意根拠の明確化
  • □ 本文に明示する送信者情報（社名・住所・連絡先）
  • □ プライバシーポリシーの掲載先URL

📝 機能仕様（4項目）

  • □ CSV項目：最小は会社名・メール。電話番号は必須？任意？業種タグは要る？
  • □ 質問パターン：二択（興味あり/なし）のみ？／日時希望や資料請求の分岐は要る？
  • □ 応答後のサンクスメール：送る？送らない？送る場合の内容は？
  • □ テスト配信の挙動：自分だけに1通？最大何通までテスト可？

🔄 運用（4項目）

  • □ 担当者割り当てロジック：自動（ラウンドロビン）／手動／指定なし
  • □ 応答から架電までの目標時間（SLA）：24時間以内？48時間以内？
  • □ キャンペーン作成権限：管理者のみ？ISメンバーも？
  • □ recipient データの保持期間：6か月？1年？永続？
