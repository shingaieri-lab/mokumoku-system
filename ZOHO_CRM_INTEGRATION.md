# Zoho CRM API連携 実装内容・設定手順書

作成日：2026-03-30

---

## 1. 連携の概要

| 方向 | 内容 |
|------|------|
| Zoho → 本ツール | リード情報（会社名・担当者・メール・メインIS担当・HPURL・ステータス）を取込 |
| 本ツール → Zoho | アクション履歴をNoteとして登録、ステータス変更を自動反映 |
| 商談確定時 → Zoho | 取引先・取引先責任者・商談を自動作成 |

---

## 2. Zoho側で必要な準備（管理者作業）

### 2-1. Zoho API Console でOAuthクライアントを作成

1. [https://api-console.zoho.jp/](https://api-console.zoho.jp/) にアクセス
2. 「Server-based Applications」タイプで新規クライアントを作成
3. Redirect URI に以下を登録する

```
https://（本ツールのドメイン）/api/zoho/callback
```

4. 作成後に表示される以下の値を控えておく
   - **Client ID**
   - **Client Secret**

### 2-2. 必要なOAuthスコープ

| スコープ | 用途 |
|---------|------|
| `ZohoCRM.modules.Leads.ALL` | リードの取得・ステータス更新 |
| `ZohoCRM.modules.Calls.CREATE` | アクション履歴を行動（コール）として登録 |
| `ZohoCRM.modules.Accounts.CREATE` | 取引先の作成 |
| `ZohoCRM.modules.Contacts.CREATE` | 取引先責任者の作成 |
| `ZohoCRM.modules.Deals.CREATE` | 商談の作成 |

### 2-3. カスタムフィールドのAPI名を確認

#### メインIS担当フィールド（リードモジュール）

「メインIS担当」フィールドのAPI名を確認する。

```
Zoho CRM管理画面
→ 設定 → カスタマイズ → モジュールとフィールド
→ リード → フィールド
→「メインIS担当」の「項目名（API名）」列を確認
例：Main_IS_Member
```

#### 商談ヘルスチェックフィールド（商談モジュール）

商談確定時に「商談日」を連携する先として、以下2フィールドのAPI名を確認する。

```
Zoho CRM管理画面
→ 設定 → カスタマイズ → モジュールとフィールド
→ 商談（Deals） → フィールド
→「商談実施日」と「完了予定日」の「項目名（API名）」列を確認
```

確認後、開発担当者に以下の形式で共有する。

| フィールド名 | API名（確認した値を記入） |
|------------|----------------------|
| 商談実施日 | （例：Shodan_Jisshi_Bi） |
| 完了予定日 | （例：Kanryo_Yotei_Bi） |

> API名が確認できたら、`server.js` の商談作成処理（`/api/zoho/create-deal`）に反映する。

### 2-4. ステータスの選択肢を確認

```
Zoho CRM管理画面
→ 設定 → カスタマイズ → モジュールとフィールド
→ リード →「リードのステータス」フィールドの選択肢一覧を確認
```

本ツールとZohoのステータス名が異なる場合、後述のマッピング設定が必要。

### 2-5. Webhook設定（リアルタイム連携・オプション）

ZohoからリードをリアルタイムでPushする場合は以下を設定する。

```
Zoho CRM管理画面
→ 設定 → 自動化 → ワークフロー → Webhook
→ 新規作成
  URL：https://（本ツールのドメイン）/api/zoho/webhook
  メソッド：POST
  モジュール：リード
  トリガー：作成時・更新時
```

### 2-6. Webhookの連携対象を特定のIS担当者に絞る（オプション）

全リードではなく、特定のIS担当者が割り当てられたリードのみを本ツールに連携したい場合、以下の2通りの方法がある。

#### 方法A：Zoho側のワークフロー条件でフィルタリング（推奨）

Zoho CRM のワークフロー設定に条件を追加することで、Webhook自体を特定のIS担当者のリードのみ発火させることができる。コード変更が不要なため、こちらを推奨する。

```
Zoho CRM管理画面
→ 設定 → 自動化 → ワークフロー → 該当のワークフローを編集
→「条件」を追加
  フィールド：メインIS担当（Main_IS_Member）
  条件：次のいずれかに等しい
  値：連携したいIS担当者の値をカンマ区切りで列挙
      例：yamada_t, suzuki_h
```

この設定により、メインIS担当が条件に一致しないリードはWebhookが発火せず、本ツールに取り込まれない。

#### 方法B：本ツール側でフィルタリング（開発作業が必要）

Zoho側の設定変更が難しい場合、本ツールのWebhook受信処理に「受け付けるIS担当者リスト」を設定するフィルタリング機能を追加することも可能。この方法は開発担当者への依頼が必要。

| 比較項目 | 方法A（Zoho側） | 方法B（本ツール側） |
|---------|--------------|----------------|
| 設定難易度 | 低（管理画面のみ） | 中（開発作業が必要） |
| コード変更 | 不要 | 必要 |
| API呼び出し数 | 少ない（Webhook自体が発火しない） | 多い（受信後に判定） |
| 推奨度 | ◎ | ○ |

## 3. 本ツール側の設定手順（管理者作業）

### 3-1. 接続設定

```
設定画面 → 「🔗 Zoho CRM連携」タブ
```

| 項目 | 内容 |
|------|------|
| データセンター | 日本（zoho.jp）を選択 |
| Client ID | 2-1で取得した値を入力 |
| Client Secret | 2-1で取得した値を入力（初回設定時のみ） |
| Redirect URI | 自動入力される（変更不要） |

### 3-2. フィールドマッピング設定

| 項目 | 内容 |
|------|------|
| メインIS担当フィールドのAPI名 | 2-3で確認した値を入力（例：`Main_IS_Member`） |

### 3-3. IS担当マッピング設定

ZohoのメインIS担当フィールドの値と、本ツールのメンバー名が異なる場合に設定する。

| Zohoの値（例） | 本ツールのメンバー名（例） |
|--------------|------------------------|
| `yamada_t` | 山田太郎 |
| `suzuki_h` | 鈴木花子 |

※ 同じ名称であれば設定不要。

### 3-4. ステータスマッピング設定（双方向）

#### 📥 Zoho → 本ツール（取込・Webhook受信時）

| Zohoのステータス（例） | 本ツールのステータス |
|----------------------|-----------------|
| New | 新規 |
| Attempted to Contact | 架電済 |
| Appointment Scheduled | 日程調整中 |
| （Zohoの正確な表記を入力） | 商談確定 |

#### 📤 本ツール → Zoho（ステータス変更時に自動反映）

| 本ツールのステータス | Zohoのステータス（例） |
|-----------------|----------------------|
| 新規 | New |
| 架電済 | Attempted to Contact |
| 日程調整中 | Appointment Scheduled |
| 商談確定 | （Zohoの正確な表記を入力） |

※ 同じ名称であれば設定不要。未設定の場合はそのままZohoに送信する。

### 3-5. OAuth認証の実行

設定保存後、「Zoho認証を開始」ボタンをクリックする。
別ウィンドウでZohoのログイン画面が開くので、CRM管理者アカウントでログインし、権限を許可する。
「認証完了」と表示されればOK。

## 4. 連携フィールド一覧

### Zoho → 本ツール（リード取込時）

| Zohoフィールド | API名 | 本ツールのフィールド |
|--------------|-------|------------------|
| 会社名 | `Company` | 会社名 |
| 姓 + 名 | `Last_Name` + `First_Name` | 担当者名 |
| メールアドレス | `Email` | メールアドレス |
| メインIS担当（カスタム） | 設定で指定 | IS担当 |
| Webサイト | `Website` | HP URL |
| リードのステータス | `Lead_Status` | ステータス |
| リードタイプ | `Lead_Type` | zoho_lead_type（商談名に使用） |

### 本ツール → Zoho（アクション履歴登録）

アクションをZohoの**行動（Calls）**として登録する。

| Zohoフィールド | API名 | 値 |
|-------------|-------|---|
| 件名 | `Subject` | `電話）インバウンド`（固定） |
| 開始日時 | `Call_Start_Time` | アクション日時（ISO 8601形式、+09:00） |
| 活動目的 | `Call_Purpose` | `追客`（固定） |
| 説明 | `Description` | アクション内容・結果・次回アクション情報 |
| 電話種別 | `Call_Type` | `Inbound`（固定） |
| 商談方法 | カスタムフィールド（要設定） | アクション方法（電話/メール/SMS/その他） |

#### 商談方法フィールドの設定

「商談方法」はZoho CRMのカスタムフィールドのため、API名を確認してから `server.js` に追加する。

```
Zoho CRM管理画面
→ 設定 → カスタマイズ → モジュールとフィールド
→ コール → フィールド
→「商談方法」の API名 を確認
```

確認後、`server.js` の `/api/zoho/push-action` 内のコメントを解除して実際のAPI名に変更する。

```javascript
// 変更前（コメントアウト）
// 例: Shodan_Houhou: methodLabel,

// 変更後（実際のAPI名に置き換え）
実際のAPI名: methodLabel,
```

### 商談確定時 → Zoho（自動作成）

| 作成するZohoレコード | 本ツールの情報から生成 |
|-------------------|--------------------|
| 取引先（Account） | 会社名、HP URL |
| 取引先責任者（Contact） | 担当者名、メールアドレス |
| 商談（Deal） | リードタイプ（`Lead_Type`）をそのまま使用、IS確度→確率（D=20%/C=40%/B=60%/A=80%）、商談日→商談実施日・完了予定日 |

#### 商談フィールドの詳細マッピング

| 本ツールのフィールド | Zoho商談フィールド | API名 | 備考 |
|-------------------|-----------------|-------|------|
| リードタイプ | 商談名 | `Deal_Name` | リード取込時に保存した値 |
| IS確度 | 確率 | `Probability` | D=20 / C=40 / B=60 / A=80 |
| 商談日 | 商談実施日 | **要確認**（2-3参照） | 商談ヘルスチェック項目 |
| 商談日 | 完了予定日 | **要確認**（2-3参照） | 商談ヘルスチェック項目 |
| −（固定値） | ステージ | `Stage` | `提案中` 固定 |

> **管理者へ：** 「商談実施日」「完了予定日」のAPI名を 2-3 の手順で確認し、開発担当者に共有してください。API名が確定次第、`server.js` に反映します。

> リードタイプはリード取込時（手動・Webhook）に自動で保存される。取込済みの旧リードには反映されないため、必要に応じて再取込を行う。

## 5. 連携の動作タイミング

| 操作 | 動作 |
|------|------|
| Zoho取込ボタン（手動） | ZohoリードIDを指定して1件取込 |
| Webhook受信（自動） | Zoho側でリード作成・更新されると本ツールに自動反映 |
| アクション記録後の🔗ボタン | そのアクション履歴をZohoに行動（コール）として登録 |
| ステータス変更 | Zoho連携済みリードのステータスをZoho側に自動反映 |
| ステータスが「商談確定」になったとき | ActionパネルのZoho商談作成ボタンが出現、押すと取引先・取引先責任者・商談を一括作成 |

---

## 6. 注意事項

- Zoho連携が有効なのは `zoho_lead_id` が設定されているリードのみ（Zohoから取込んだリード、またはWebhookで連携されたリード）
- Client Secretはセキュリティのためサーバー側のみに保存され、画面には表示されない
- アクセストークンは1時間で失効するが、リフレッシュトークンで自動更新される（再認証不要）
- 商談作成は1リードにつき1回のみ推奨（重複作成の防止機能はないため手動で確認）
