# IS進捗管理ツール（ダンドリワーク）UXレビュー

レビュー日：2026-03-23　／　最終更新：2026-03-25（バグ修正対応：2026-03-25、UIコントラスト修正：2026-03-25）

---

## このツールは何をするもの？

営業チームが「どの会社に連絡したか」「次はいつ連絡するか」を管理するツール。
AIが通話メモを読んで次の行動を提案したり、Googleカレンダーの空き時間を自動で探したりできる。

---

## 1. 使いやすさ（クリック数）

### ✅ うまくいっているところ

- リード新規登録：**3クリック**で完了
- 設定画面を開く：**3クリック**

### ❌ 改善が必要なところ

**候補日を探してメールを送るまでの流れが長すぎる（8〜10クリック）**

```
①候補日ページを開く
②検索条件を入力する
③「検索」ボタンを押す
④空き時間を3つ選ぶ
⑤「メールに使う」ボタンを押す　← ページが切り替わる
⑥メールページでもう一度リードを選ぶ　← ❌ さっき選んでいるのに！
⑦テンプレートを選ぶ
⑧「Gmail下書き保存」を押す
```

~~**⑥がムダ。** 候補日ページで選んだリード情報をメールページに引き継ぐべき。~~

← **対応済み**：リード情報・テンプレートが自動引き継ぎされるよう修正（コミット：`3845520`）

## 2. 見た目の一貫性

### ✅ AIページだけ見た目が全然違う　← **対応済み**

| ページ | 背景・テキスト |
|--------|--------------|
| ダッシュボード・リード管理など | 白背景＋緑文字 |
| **AIページだけ** | 真っ暗な紺色背景＋白文字 |

「AIアシスタント」タブを押した瞬間に別のアプリのような見た目になる。
初めて使うユーザーが「バグ？」と感じる可能性がある。
他のページと同じ白系背景に統一、またはわずかな差にとどめるのが望ましい。
← **対応済み**：ライトグリーン系に統一（コミット：`aed959f`）

### ✅ ナビのラベルが小さすぎる　← **対応済み（10pxに修正）**

| 場所 | フォントサイズ |
|------|-------------|
| デスクトップ サイドバー | **9.5px** |
| スマホ ボトムナビ | **8px** |

一般的なWebの最小推奨は12px。8pxはほぼ読めない大きさ。
アイコンで判断させる意図であれば、ラベルを非表示にするか、最低11〜12pxに拡大すべき。
← **対応済み**：10pxに拡大（コミット：`ac443b0`）

## 3. コードの問題（今後の開発に影響）

### ✅ Gmail下書き保存のコードが2か所に重複（約80行）　← **対応済み**

`EmailPage` の `saveGmailDraft` と `AIPage` の `saveAiGmailDraft` を共通関数に統合。
（対応コミット：`998803f`）

### ✅ 祝日リストが2か所に重複定義　← **対応済み**

グローバルの `JP_HOLIDAYS` を AIPage でも参照するよう修正。祝日追加が1か所で完結するようになった。
（対応コミット：`998803f`）

### ✅ メールテンプレートの初期値が2か所で微妙にズレている　← **対応済み**

テンプレート初期値を1か所に統一。
（対応コミット：`998803f`）

### ❌ 単一HTMLファイル（行数は更新後も大規模）

全ロジック・スタイル・テンプレートが1ファイルに集中しており、機能追加・バグ修正の難易度が高い。
長期的にはコンポーネントごとにファイル分割を検討する。

## 4. セキュリティの問題

## 5. AIページ：解析後の導線が不明

解析→保存した後、次に何をすればいいかの案内がない。

**現状：**
```
解析する → 保存完了！ ← ここで終わり
```

**改善案：**
```
解析する → 保存完了！
           → 「📧 フォローメールを送る」ボタン
           → 「📅 商談候補日を探す」ボタン
```

## 6. 設定の初回セットアップが難しすぎる

Google Calendar 連携を開始するまでの手順：

1. Google Cloud Console でプロジェクト作成
2. Calendar API を有効化
3. APIキーを取得
4. OAuth クライアントID を作成
5. 承認済みオリジンの設定
6. 各担当者のカレンダーID を収集
7. アプリの設定画面に全部入力

現状は文字の説明だけで案内しており、IT知識のないユーザーには高いハードル。

← **対応済み**：Gemini・Gmail・Google カレンダーのAPI設定ウィザードを実装（コミット：`bf7545e`）

## 7. 確認ダイアログがブラウザのポップアップ

削除確認に `window.confirm`（ブラウザ標準のグレーのポップアップ）を使用。
アプリのデザインと統一感がなく、ユーザーが戸惑う。

- **改善案：** アプリ内モーダル（ダイアログ）で確認を取る

## 10. 新規発見：UI/UX の問題（2026-03-25）

### 🔴 高

#### モバイル対応の不完全さ（メール・AI・設定ページ）

`isMobile` フラグでレイアウトを切り替えているが、一部の画面でカバーが不完全。

- AIページ左パネル：`width:380`（固定）→ モバイルで全幅を圧迫
- メールテンプレート：`gridTemplateColumns:"240px 1fr"`（固定）
- 設定ページのタブ：`width:"60vw"`（固定）→ モバイルでオーバーフローの可能性

**改善案：** ブレークポイントで固定幅をすべて可変に。ボタンパディングをモバイルで拡大。

---

### 🟠 中

#### 削除・編集ボタンが初期状態で非常に薄い（設定ページ）

`opacity:0.3` で表示され、ホバー時だけ `opacity:1` になる。タッチデバイスではホバーが発生しないため、ボタンの存在に気づけない。

**改善案：** 常時 `opacity:0.7` 以上に変更。

#### 成功メッセージの消失が速すぎる（全体）

「✅ 下書き保存済」が3秒、「保存しました ✓」が2秒で自動消失。読む前に消えるケースがある。

**改善案：** 5〜8秒に延長、または ✕ ボタンで手動 Close。

#### エラーメッセージの視認性が低い（AIページ・フォーム全般）

エラー背景が薄く（`#ef444416`）、複数エラー時は一括 `alert()` で表示されるため、どのフィールドに問題があるか分かりにくい。

**改善案：** 背景を濃くしアイコンを付ける。各フィールドに個別エラーを表示。

#### ボタンが小さすぎてタッチ操作しにくい（テンプレート・設定リスト）

`padding:"3px 10px"`, `fontSize:12` で高さ約20px。WCAG 推奨の最小タッチ領域は 44×44px。

**改善案：** 最低 `padding:"6px 12px"`（高さ32px以上）に統一。

#### バルク操作の完了通知がない（リード管理）

複数リードのインポート・削除完了時に視覚的フィードバックがなく、処理結果が分かりにくい。

**改善案：** 画面右下にトースト通知（例：「10件追加しました ✓」）を実装。

#### ローディング表示が統一されていない（全体）

AI解析中はスピナーが表示されるが、Gmail保存・Googleタスク作成などはテキスト変化のみ。

**改善案：** 全ての非同期処理で同じスピナーを使用。
