## 1. このツールでできること

**IS進捗管理ツール**は、インサイドセールス（電話やメールで営業するチーム）の仕事をサポートするための管理システムです。

### 主な機能一覧

| 機能 | 説明 |
|------|------|
| リード管理 | 見込み客（お問い合わせをくれた会社）の情報を登録・管理する |
| アクション履歴 | 電話やメールなど、顧客への対応記録を残す |
| AIアシスタント | 通話メモをAIが読んで、次のアクションや送るメールを自動で提案する |
| メールテンプレート | よく使うメール文章を保存しておき、すぐに使い回せる |
| 日程調整 | Googleカレンダーと連携して、空き時間を自動で探す |
| ダッシュボード | 今日やるべきことや、全体の進捗をひと目で確認できる |
| マスター設定 | チームメンバーや媒体（ポータルサイトなど）の設定を管理する |

## 2. アカウントの種類と権限

このツールには **2つの軸** でアカウントの種類が決まります。

---

### 軸① ロール（役割）：管理者 か メンバー か

> **ロール（役割）とは？** そのアカウントがシステム全体に対して持つ権限のレベルです。

| ロール | 説明 |
|--------|------|
| **管理者** | チームの設定・アカウント管理など、すべての機能を使える |
| **メンバー** | 日常の営業活動（リード対応・AI活用・メール作成など）を行う |

---

### 軸② IS担当フラグ：IS担当 か IS担当以外（閲覧のみ）か

> **IS担当フラグとは？** インサイドセールスの担当者かどうかを示す設定です。ロールとは独立して設定します。

| 設定 | 説明 |
|------|------|
| **IS担当** | リード対応・アクション記録など、通常の営業業務ができる |
| **IS担当以外（閲覧のみ）** | AIアシスタント・メール・カレンダーは利用できる。リードの追加・編集・削除とアクション記録はできない |

---

### 3種類のアカウント組み合わせ

実際には以下の3パターンで運用します：

#### パターン① 管理者 ＋ IS担当
チームリーダーや管理担当者向け。すべての機能が使えます。

#### パターン② メンバー ＋ IS担当
一般のインサイドセールス担当者向け。日常の営業業務を行います。

#### パターン③ 閲覧のみ（IS担当以外）
営業以外のスタッフ向け。リードの閲覧・AIアシスタント・メール・カレンダーは使えます。リードの追加・編集・削除やアクション記録はできません。

---

### 権限早見表

> ○ = 使える　△ = 一部のみ　✕ = 使えない

| できること | 管理者＋IS担当 | メンバー＋IS担当 | 閲覧のみ |
|-----------|:------------:|:---------------:|:-------:|
| ダッシュボードの閲覧 | ○ | ○ | ○ |
| リード一覧の閲覧 | ○ | ○ | ○ |
| リードの追加・編集・削除 | ○ | ○ | ✕ |
| アクション履歴の記録・編集・削除 | ○ | ○ | ✕ |
| AIアシスタントの利用 | ○ | ○ | ○ |
| メールテンプレートの作成・利用 | ○ | ○ | ○ |
| 日程調整（カレンダー連携） | ○ | ○ | ○ |
| 営業設定の変更 | ○ | △（自分の設定のみ） | ✕ |
| API設定の確認・変更 | ○ | ○ | ○ |
| **リード管理設定（マスター）** | **○** | **✕** | **✕** |
| **ポータルサイト設定（マスター）** | **○** | **✕** | **✕** |
| **営業担当メンバーの管理** | **○** | **✕** | **✕** |
| **アカウント管理（ユーザー追加・削除）** | **○** | **✕** | **✕** |
| **招待コードの発行** | **○** | **✕** | **✕** |
| **他ユーザーのパスワードリセット** | **○** | **✕** | **✕** |
| **ログインロックの解除** | **○** | **✕** | **✕** |

> 太字の行は **管理者のみ** が使える機能です。

---

### IS担当として選択される条件

リードに「担当IS」を割り当てる際の候補リストには、**IS担当フラグがONになっているアカウント** だけが表示されます。閲覧のみのアカウントは候補に表示されません。

## 3. ログイン・アカウント登録

### ログイン方法

どのアカウント種別（管理者・メンバー・閲覧のみ）でも、ログイン手順は同じです。

1. ブラウザ（ChromeやSafariなど）でツールのURLを開く
2. **ユーザーID**（メールアドレス）と**パスワード**を入力する
3. 「ログイン」ボタンをクリックする

> **ポイント：** 5回連続でパスワードを間違えると、セキュリティのため15分間ログインできなくなります。ログイン解除は管理者のみが行えます。

---

### サインアップ（新規アカウント登録）の流れ

#### ケース①：初めてツールを使い始めるとき（最初の1人目）

招待コードは不要です。最初に登録したアカウントは自動的に **管理者** になります。

1. ログイン画面で「新規登録」を選ぶ
2. ユーザーID・名前・メールアドレス・パスワードを入力する
3. 「IS担当」チェックボックスで自分がIS担当かどうかを選ぶ
4. 「登録」ボタンをクリックする

#### ケース②：2人目以降のメンバーを追加するとき

管理者から **招待コード** を受け取ってから登録します。

1. 管理者がアカウント管理画面で「招待コードを発行」する
2. 発行されたコードを新しいユーザーに共有する
3. 新しいユーザーがログイン画面で「新規登録」を選ぶ
4. **招待コード** を入力する
5. ユーザーID・名前・メールアドレス・パスワードを入力する
6. 「IS担当」チェックボックスを選ぶ
7. 「登録」ボタンをクリックする

> **招待コードとは？** 許可された人だけが登録できるように、管理者が発行する使い捨ての番号です。**24時間で使えなくなります。** 期限切れの場合は管理者に再発行を依頼してください。

> **注意：** 登録直後のロールは一時的に「管理者」になります。管理者がアカウント管理画面で「メンバー」に変更してください。

---

### パスワードを忘れたとき

管理者に連絡してください。管理者がパスワードをリセットできます（自分自身のパスワードは自分で変更可能です）。

## 4. ダッシュボード（トップ画面）

ログイン後に最初に表示される画面です。営業活動の全体像をひと目で確認できます。

### 表示される情報

#### KPIカード（重要な数字の概要）
> KPIとは「Key Performance Indicator」の略で、目標達成度を測る重要な数値のことです。

| カード名 | 説明 |
|----------|------|
| 総リード数 | 登録されている見込み客の合計数 |
| 商談数 | 商談（打ち合わせ）まで進んだ件数 |
| 受注予測 | 受注になりそうな案件の合計金額の見込み |
| 本日のフォロー数 | 今日対応が必要なリストの件数 |

#### コンバージョン率（転換率）
> コンバージョン率とは、問い合わせが来てから実際に受注になるまでの確率のことです。

媒体（ポータルサイト、HP、電話など）ごとに、どれだけ成果が出ているかが確認できます。

#### 本日のフォローリスト
今日アクション（電話・メールなど）をする予定のリードが自動的にリストアップされます。リード名をクリックすると詳細画面が開きます。

---

## 5. リード管理（見込み客の管理）

**リード**とは、自社サービスに興味を持っている見込み客（会社・担当者）のことです。

### リード一覧を見る

メニューから「リード」を選ぶと、登録されているリードの一覧が表示されます。

- **検索・絞り込み：** 会社名、担当者名、ステータスなどで絞り込めます
- **並べ替え：** 登録日や次回アクション日などで並べ替えられます

### 新しいリードを登録する

1. 「新規登録」ボタンをクリックする
2. 以下の情報を入力する

| 項目 | 説明 | 必須 |
|------|------|------|
| 会社名 | リードの会社名 | ○ |
| 担当者名 | 連絡先の方の名前 | |
| 電話番号 | 連絡先電話番号 | |
| メールアドレス | 連絡先メール | |
| ステータス | 現在の進捗状況（例：初回コール、商談中、受注など） | ○ |
| 媒体・ソース | どこから問い合わせが来たか（例：ポータルサイト、HP） | |
| 担当IS | 担当するインサイドセールス担当者 | |
| 担当営業 | 担当する営業担当者 | |
| 商談日 | 商談の予定日・実施日 | |
| MQLステータス | マーケティング観点での見込み度（Marketing Qualified Lead）| |

3. 入力後「保存」ボタンをクリックする

### リードを編集する

リスト内のリード名をクリックすると詳細画面が開きます。「編集」ボタンを押して内容を変更し、「保存」で確定します。

### リードを削除する

詳細画面の「削除」ボタンから削除できます。削除したデータは元に戻せないため、ご注意ください。

### ステータス（進捗）について

リードの進捗状況を示します。代表的なステータスの例：

| ステータス | 意味 |
|-----------|------|
| 新規 | まだ最初の連絡をしていない |
| 初回コール済み | 最初の電話をした |
| 商談中 | 打ち合わせが始まっている |
| 受注 | 契約が決まった |
| 失注 | 契約に至らなかった |
| ナーチャリング | 今すぐではないが将来的な見込みあり |

> ステータスの種類は管理者がカスタマイズできます。

---

## 6. アクション履歴（対応記録）

リードごとに、電話やメールなどの対応記録（アクション）を残します。

### アクションを記録する

1. リード詳細画面を開く
2. 「アクション追加」ボタンをクリックする
3. 以下を入力する

| 項目 | 説明 |
|------|------|
| アクション種別 | 電話 / メール / SMS / その他 |
| 結果 | つながった / 不在 / 折り返し待ち / 留守電 など |
| 対応メモ | 通話の内容や気づいたことを自由に記録 |
| 次回アクション日 | 次にフォローする予定の日付 |

4. 「保存」をクリックする

> **ポイント：** 次回アクション日を設定しておくと、その日にダッシュボードの「本日のフォローリスト」に自動で表示されます。

### 過去のアクション履歴を確認する

リード詳細画面の下部に、過去のアクション一覧が日付の新しい順で表示されます。

## 7. AIアシスタント

**Google Gemini AI**（Googleが開発した人工知能）を活用して、営業活動をサポートする**専用ページ**です。メニューの「AIアシスタント」から開きます。

> **AIとは？** 大量のデータから学習し、人間のような判断や文章生成ができるコンピュータープログラムです。

### AIアシスタントでできること

- 通話内容のメモを読んで**アクション内容（今回の対応）を自動要約**する
- **次回アクションの日時・種別・メモ**を提案する
- 次回架電時の**トークポイント（話すべきポイント）を最大4つ提案**する
- 見込み客の**興味度（高・中・低）を評価**する
- 送るべき**メール件名と本文を自動生成**する

### 使い方

1. メニューから「AIアシスタント」ページを開く
2. 以下を入力する

| 入力項目 | 説明 |
|----------|------|
| リード選択 | 対応した見込み客を選ぶ（過去のアクション履歴も自動でAIに渡される） |
| 対応メモ | 今回の電話・メールの内容を自由に記録する |
| 対応日時 | 対応した日付と時間 |
| アクション種別 | 電話 / メール / SMS / その他 |
| 対応結果 | 取次 / 折り返し待ち / 留守電 など |

3. 「解析」ボタンをクリックする（AIが分析を開始する）
4. 以下の結果が表示される：
   - **今回のアクション要約**
   - **次回アクション日・時間・メモ**（提案は営業時間9〜18時に自動補正される）
   - **フォローアップトークポイント**（最大4つ）
   - **興味レベル**（高・中・低）
   - **メール件名と本文**（担当者名・会社名・署名が自動挿入済み）
5. 「アクションを保存」ボタンをクリックすると、対応記録がリードに追加され、次回アクション日時も自動で更新される
6. メール文章は「Gmailに下書き保存」でGmailに直接送れる

> **ポイント：** 別のページに移動しても、AIの解析結果や入力内容はそのまま保持されます。ページに戻ると続きから作業できます。

> **注意：** AIの提案はあくまで参考です。内容を確認してから使ってください。

### AIを使うための準備（初回のみ・管理者または各自が設定）

- **Google Gemini APIキー**を設定画面の「API設定」に登録する必要があります
- APIキーはGoogle AI Studio（https://aistudio.google.com）から取得できます

## 8. メールテンプレート機能

よく使うメール文章を「テンプレート（雛形）」として保存しておき、リードに合わせて素早くメールを作成できます。メニューの「メールテンプレート」から開きます。

### 画面の構成

| エリア | 説明 |
|--------|------|
| 左側：テンプレート一覧 | 保存済みのテンプレートが並ぶ。ドラッグ＆ドロップで並び順を変えられる |
| 中央：差し込み変数 | リード選択と変数の入力欄 |
| 右側：メール本文プレビュー | 変数が反映された完成形。直接編集も可能 |

> テンプレートは自由に追加・編集・削除できます。

### 使えるすべての変数

テンプレートの件名・本文に以下の変数を入れると、対応する情報が自動で埋め込まれます。

| 変数 | 埋め込まれる内容 | 自動入力 |
|------|----------------|---------|
| `{{担当者名}}` | 担当者のフルネーム | リード選択時に自動 |
| `{{担当者苗字}}` | 担当者の苗字のみ | リード選択時に自動抽出 |
| `{{会社名}}` | 会社名 | リード選択時に自動 |
| `{{送信者名}}` | ログイン中のユーザー名 | 常に自動 |
| `{{候補日時}}` | カレンダーで選んだ候補日時 | 日程調整ツールから自動 |
| `{{商談月}}` | 商談の月（例：3） | 手動または商談確定リードから自動 |
| `{{商談日}}` | 商談の日（例：20） | 手動または商談確定リードから自動 |
| `{{商談曜日}}` | 商談の曜日（例：金） | 月・日から自動計算 |
| `{{商談時}}` | 商談の時間（例：10） | 手動または商談確定リードから自動 |
| `{{商談分}}` | 商談の分（例：00） | 手動または商談確定リードから自動 |
| `{{商談担当}}` | 商談の担当営業名 | 手動または商談確定リードから自動 |

> 設定画面の「アカウント管理」で**署名（メール末尾の挨拶文）**を登録しておくと、メール本文の末尾に自動で追加されます。

### テンプレートの種類フラグ（作成・編集時の設定）

テンプレートを作成・編集するとき、以下の2つのチェックボックスがあります。

| フラグ | 説明 |
|--------|------|
| `{{候補日時}}` を使用する | 日程調整テンプレートとして扱われる。カレンダーで選んだ候補日時が自動で入る |
| `{{商談月}}`〜`{{商談担当}}` を使用する | 商談確定テンプレートとして扱われる。このテンプレートでGmail下書き保存すると、リードのステータスが**「商談確定」に自動更新**される |

### テンプレートを使ってメールを作成する手順

1. 左側のリストからテンプレートを選ぶ
2. 「リード選択」で対象の見込み客を検索・選択する
   → 担当者名・会社名・宛先メールアドレス・商談情報が自動で入力される
3. 右側のプレビューでメール内容を確認する（必要なら直接編集も可能）
4. 「↩ リセット」ボタンで編集前の状態に戻すこともできる
5. 「📨 Gmailに下書き保存」または「📋 全文コピー」を選んで完成

### テンプレートを追加・編集する

- 「＋ テンプレート追加」ボタンで新しいテンプレートを作成する
- テンプレート一覧の鉛筆（✏️）アイコンで編集する
- テンプレート一覧のゴミ箱（🗑）アイコンで削除する
- テンプレートは左端の「⠿」マークをドラッグして並び順を変えられる

## 9. 日程調整（候補日ツール）

Googleカレンダーと連携して、複数のチームメンバーの空き時間を自動で探します。メニューの「候補日ツール」から開きます。

> **注意：** この機能を使うには、管理者がGoogleカレンダーAPIキーを設定し、各メンバーのカレンダーIDが登録されている必要があります。

### 使い方

1. 「候補日ツール」ページを開く
2. 以下の検索条件を設定する：

| 設定項目 | 初期値 | 説明 |
|----------|--------|------|
| 対象メンバー | - | 空き時間を調べたいチームメンバー |
| 候補期間（開始〜終了） | 本日〜14日後 | 検索する期間 |
| 所要時間 | 60分 | 打ち合わせの長さ |
| 稼働時間 | 09:00〜18:00 | 候補として提示する時間帯 |
| 前バッファ | 30分 | 商談開始前に空けておく時間 |
| 後バッファ | 30分 | 商談終了後に空けておく時間 |
| 対象曜日 | 月〜金 | 候補として提示する曜日 |
| 祝日を含む | OFF | ONにすると祝日も候補に含める |
| 除外時間帯 | 12:00〜13:00 | 候補として提示しない時間帯（複数設定可能） |

3. 「空き時間を検索」ボタンをクリックする
4. 空きスロット（時間枠）の一覧が表示される
   - 各スロットには**空いているメンバー名**が表示される
   - 複数メンバーを選択した場合、少なくとも1人が空いていれば表示される
5. 候補にしたいスロットを選択する（**最大3つまで**）
6. 選択後、以下のどちらかで次のステップへ：

| ボタン | 動作 |
|--------|------|
| 「メール作成へ」 | メールテンプレートページへ移動し、`{{候補日時}}`に選んだ日時が自動挿入される |
| 「カレンダー登録」 | 選択した候補日時をGoogleカレンダーにイベントとして登録する（Gmail OAuthが必要） |

> **スロットの単体コピー：** 各スロット右側のコピーボタンをクリックすると、その日時をクリップボードにコピーできます。

## 10. マスター設定（管理者向け）

チーム全体の基本情報を設定・管理する画面です。**管理者権限**を持つアカウントのみ操作できます。

### 設定できる項目

#### チームメンバー
- 担当ISや担当営業として選択できるメンバーを登録・管理する
- 表示用の色（カラー）を設定できる

#### ポータルサイト・媒体
- リードの流入元（どこから問い合わせが来たか）を管理する
- 掲載費用や課金タイプ（月額・成果報酬など）を設定できる
- 媒体ごとのコスト・獲得単価をダッシュボードで確認できる

#### ステータス
- リードの進捗ステータスをカスタマイズできる
- 表示名・色を自由に設定できる

#### ソース（問い合わせ元）
- ポータル / HP / 電話 / 紹介 など、問い合わせの種類を管理する

#### API設定
- **Google Gemini APIキー：** AIアシスタント機能を使うために必要
- **Gmail OAuth クライアントID：** Gmailの下書き保存機能を使うために必要
- **Google Calendar APIキー：** 日程調整機能を使うために必要

> **APIキーとは？** 外部サービス（GoogleのAIやカレンダーなど）を利用するための「合言葉」のようなものです。

## 11. アカウント管理（管理者向け）

ユーザーアカウントの追加・削除・パスワード管理を行います。この画面は **管理者 ＋ IS担当** のアカウントのみ操作できます。

### 新しいユーザーを招待する

1. アカウント管理画面を開く
2. 「招待コードを発行」ボタンをクリックする
3. 表示された招待コードを新しいユーザーに共有する
4. ユーザーは招待コードを使って自分でアカウントを作成する（→ 詳細は [3章 サインアップ](#3-ログインアカウント登録) を参照）

> **招待コードは24時間で無効になります。** 期限が切れた場合は新しいコードを発行してください。

### ロールと IS担当フラグを変更する

登録直後のアカウントは一時的に「管理者」になっています。必要に応じてロールを変更してください。

1. アカウント一覧から対象のユーザーを選び「編集」をクリックする
2. **ロール** のドロップダウンで「管理者」または「メンバー」を選ぶ
3. **IS担当** チェックボックスで ON / OFF を切り替える
4. 「保存」をクリックする

| 設定の組み合わせ | 対象のメンバー像 |
|----------------|----------------|
| 管理者 ＋ IS担当 ON | チームリーダー・管理担当者 |
| メンバー ＋ IS担当 ON | 一般のインサイドセールス担当者 |
| （管理者 or メンバー）＋ IS担当 OFF | 閲覧のみのスタッフ |

### パスワードをリセットする

1. アカウント一覧から対象のユーザーを選ぶ
2. 「パスワードリセット」ボタンをクリックする
3. 新しい仮パスワードが表示されるので、ユーザーに伝える

### ログインロックを解除する

5回以上パスワードを間違えたユーザーは15分間ログインできなくなります。すぐに解除したい場合は：

1. アカウント管理画面の「ログインロック一覧」を開く
2. 対象のユーザーの「ロック解除」ボタンをクリックする

### アカウントを削除する

アカウント一覧から対象ユーザーの「削除」ボタンをクリックします。削除すると元に戻せません。

## 12. よくある質問・困ったときは

### Q. ログインできない

- パスワードを確認してください
- 5回以上間違えた場合は15分待ってから再試行してください
- それでもログインできない場合は管理者にパスワードリセットを依頼してください

### Q. AIアシスタントが動かない

- 管理者がGemini APIキーを設定しているか確認してください
- インターネット接続を確認してください

### Q. Gmailに下書きが保存されない

- 管理者がGmail OAuth設定をしているか確認してください
- ブラウザのポップアップブロックが有効になっている場合、許可が必要です

### Q. カレンダーの空き時間が表示されない

- 管理者がGoogle Calendar APIを設定しているか確認してください
- マスター設定でカレンダー対象のメンバーIDが正しく登録されているか確認してください

### Q. 登録したデータが消えた

- 保存ボタンを押さずに画面を閉じた可能性があります
- 変更後は必ず「保存」ボタンをクリックしてください

### Q. 招待コードの期限が切れた

- 管理者に新しい招待コードの発行を依頼してください
