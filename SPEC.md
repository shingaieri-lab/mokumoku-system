# IS進捗管理ツール 仕様書

**ツール名：** IS進捗管理（ダンドリワーク）
**バージョン：** 1.0.0
**作成日：** 2026-03-23

---

## 1. ツール概要

営業チームのインサイドセールス（IS）活動を管理するWebアプリケーション。

- 顧客リード（見込み客）の進捗状況を一元管理する
- AI（Google Gemini）が通話メモを解析して次のアクションを提案する
- Googleカレンダーと連携して商談候補日を自動検索する
- Gmailの下書きをワンクリックで作成する

---

## 2. 技術スタック

| 区分 | 技術 |
|------|------|
| フロントエンド | React 18（CDN経由、ビルド不要） |

| メール | Gmail API（OAuth 2.0） |

## 3. セットアップ手順

### 3-1. 必要なアカウント

- **Vercel**アカウント（ホスティング・KV）
- **Google Cloud Console**アカウント（Calendar API・Gmail API・Gemini API）

### 3-2. Vercelへのデプロイ

```bash
# 1. リポジトリをクローン
git clone <リポジトリURL>
cd <プロジェクト名>

# 2. Vercel CLIをインストール
npm install -g vercel

# 3. デプロイ
vercel deploy --prod
```

### 3-3. Vercel KVの設定

1. Vercelダッシュボード → Storage → 「Create Database」→ KVを選択
2. プロジェクトにKVをリンク
3. 環境変数 `KV_URL`、`KV_REST_API_URL`、`KV_REST_API_TOKEN` が自動設定される

### 3-4. Google API の設定（アプリ内から設定可）

デプロイ後、アプリにサインアップしてログインし、設定画面から以下を入力する。

#### Gemini API（AIアシスタント用）

1. [Google AI Studio](https://aistudio.google.com/) でAPIキーを取得
2. アプリ設定 → 「APIキー設定」→ Gemini API Key に貼り付け

#### Google Calendar API（候補日検索用）

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. 「Google Calendar API」を有効化
3. 認証情報 → APIキーを作成
4. 各メンバーのGoogleカレンダーID（メールアドレス形式）を収集
5. アプリ設定 → 「カレンダー設定」に入力

#### Gmail API（下書き保存用）

1. Google Cloud Console → 認証情報 → OAuthクライアントIDを作成（種類：ウェブアプリケーション）
2. 「承認済みJavaScriptオリジン」にデプロイ先URL（例：`https://example.vercel.app`）を追加
3. アプリ設定 → 「APIキー設定」→ Gmail Client ID に貼り付け

## 4. 機能一覧

### 4-1. 認証

| 機能 | 説明 |
|------|------|
| サインアップ | 初回のみID・パスワードで管理者アカウントを作成 |
| ログイン | ID・パスワードで認証。セッショントークンを発行（有効期限30日） |
| ログアウト | セッションを無効化 |

> **注意：** 現バージョンはパスワードが平文で保存されます。社内限定・テスト用途での使用を推奨します。

### 4-2. ダッシュボード

月次の営業KPIをカード形式で表示する。月セレクタで過去月のデータも参照可能。カードをクリックすると該当条件でフィルタされたリード一覧に遷移する。

| KPIカード | 説明 |
|----------|------|
| 総反響数 | 選択月に登録したリード数。MQL件数・率を補足表示 |
| 商談設定数 | ステータスが「日程調整中」または「商談確定」のリード数。商談化率を補足表示 |
| 課金見込み | ポータルサイト経由リードの課金合計金額（対象外申請済は除外） |
| 本日追客 | 自分担当の次回アクション日が今日のリード数。期限切れ件数を補足表示 |

画面下部には以下のパネルも表示される。

- **流入元別商談化実績** — 各流入元の反響数・商談数・MQL数・商談化率をプログレスバー付きで表示
- **ステータス分布** — 各ステータスの件数と割合
- **ポータルサイト課金管理** — ポータルサイト別の課金件数・金額・獲得単価・対象外申請済件数

### 4-3. リード管理

見込み客（リード）の一覧表示・登録・編集・削除を行う。

#### リード情報フィールド

| フィールド | 説明 |
|-----------|------|
| 会社名 | 必須 |
| 担当者名 | 任意 |
| 電話番号 | 任意 |
| メールアドレス | 任意 |
| 流入元 | マスター設定で定義したソース |
| ポータルサイト名 | 流入元がポータルサイト系の場合に選択 |
| 請求種別 | ポータルサイトの請求タイプ |
| ステータス | マスター設定で定義したステータス |
| IS担当者 | アカウント一覧から選択 |
| 営業担当者 | マスター設定のメンバーから選択 |
| MQL | MQL / 非MQL の選択 |
| メモ | 自由記述 |
| 次回アクション日 | 次に連絡する予定日 |
| 次回アクション時刻 | 次に連絡する予定時刻 |
| 次回アクションメモ | 次回の連絡内容メモ |
| 商談日・商談時刻 | 商談確定時に入力 |
| 商談担当者 | 商談に出席する営業担当者 |

#### アクション履歴

リードごとに架電・メール・SMS等の活動履歴を記録できる。

| フィールド | 説明 |
|-----------|------|
| アクション種別 | 電話 / メール / SMS / その他 |
| アクション結果 | 取次 / 不在 / 不通 / 折電 / 送信済 / その他 |
| 実施日時 | 自動入力（手動変更可） |
| メモ | 通話内容のメモ |

### 4-4. AIアシスタント

通話メモを入力すると、Google Gemini APIが解析して以下を自動生成する。

| 出力項目 | 説明 |
|---------|------|
| アクションサマリー | 架電内容の要約（2〜3文） |
| アクション種別 | 自動判定 |
| アクション結果 | 自動判定 |
| 次回アクション日 | 最適な架電間隔を計算して提案 |
| 次回アクション時刻 | 提案時刻 |
| 次回アクションメモ | 次回連絡時のポイント |
| フォローアップトークポイント | 次回使えるトーク例 |
| フォローメール件名・本文 | そのままコピーして使えるメール文章 |
| 興味度 | 高 / 中 / 低 |
| Zoho用サマリー | CRM転記用の詳細サマリー |

**解析結果はワンクリックでリードに保存でき、Gmailの下書きも同時に作成できる。**

### 4-5. メールテンプレート

差し込み変数を使ったメールテンプレートを管理する。

#### 差し込み変数一覧

| 変数 | 内容 |
|------|------|
| `{{担当者名}}` | リードの担当者名（フルネーム） |
| `{{担当者苗字}}` | 担当者名の苗字部分（自動抽出） |
| `{{会社名}}` | リードの会社名 |
| `{{送信者名}}` | ログインユーザーの名前（自動入力） |
| `{{候補日時}}` | 候補日ツールで選択した日程（改行区切り） |
| `{{商談月}}` `{{商談日}}` `{{商談曜日}}` | 商談確定日の各要素 |
| `{{商談時}}` `{{商談分}}` | 商談確定時刻 |
| `{{商談担当}}` | 商談担当者名 |

#### テンプレートの種類

| 種類 | 説明 |
|------|------|
| 通常 | 変数を差し込んでコピーするだけ |
| 日程調整 | `{{候補日時}}` を使用。候補日ツールと連携 |
| 商談確定 | 商談日時フィールドが表示される。下書き保存時にリードの商談情報を自動更新 |

### 4-6. 商談候補日検索

メンバーのGoogleカレンダーを参照して空き時間を自動検索する。

**設定できる条件：**
- 検索期間（開始日・終了日）
- 検索対象のメンバー（複数選択可）
- 1スロットの時間（分）
- バッファ（前後の余裕時間）
- 対象外の時間帯（例：12:00〜13:00は昼休みとして除外）
- 対象外の曜日

**出力：** 条件に合致する候補スロット一覧。最大3つ選択してメールテンプレートに転送できる。

### 4-7. マスター設定

管理者が各種マスターデータを管理する。

| 設定項目 | 説明 |
|---------|------|
| 営業メンバー | 営業担当者の名前一覧 |
| ポータルサイト | 媒体名・請求種別・単価・ひもづく流入元の設定 |
| 流入元 | リードの流入チャネル名とアイコン |
| ステータス | 営業進捗のステータス名と表示色 |

### 4-8. アカウント管理（管理者のみ）

| 機能 | 説明 |
|------|------|
| アカウント一覧表示 | 登録済みユーザーの一覧 |
| アカウント追加・編集 | 名前・ID・パスワード・色・署名・IS担当フラグの設定 |
| アカウント削除 | 管理者のみ実行可能 |

#### アカウントフィールド

| フィールド | 説明 |
|-----------|------|
| 名前 | 表示名 |
| ID | ログインID |
| パスワード | ログインパスワード |
| 色 | カレンダー・リスト上の識別色 |
| 署名 | メール送信時に自動付与される署名 |
| Gemini APIキー | 個人用APIキー（設定なしの場合は共通キーを使用） |
| Gmail Client ID | 個人用Client ID |
| IS担当フラグ | ONにするとIS担当者として候補に表示される |

## 5. API仕様

すべてのAPIエンドポイント（ログイン・サインアップ以外）はリクエストヘッダーに `x-session-token` が必要。

### 認証

| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/api/login` | ログイン。`{ id, password }` を送信してトークンを取得 |
| POST | `/api/signup` | 新規アカウント作成。`{ id, password, name, ... }` を送信 |
| POST | `/api/logout` | ログアウト。セッションを削除 |

### データ取得・保存

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/data` | 全データを一括取得（ログイン後の初期ロード） |
| POST | `/api/leads` | リードデータを全件保存（配列） |
| POST | `/api/accounts` | アカウントデータを全件保存（配列） |
| POST | `/api/master-settings` | マスター設定を保存 |
| POST | `/api/ai-config` | AI設定（APIキー等）を保存 |
| POST | `/api/gcal-config` | カレンダー設定を保存 |
| POST | `/api/email-tpls` | メールテンプレートを全件保存（配列） |
| POST | `/api/reset` | 全データを削除（管理者のみ） |

### レスポンス形式

**成功時**
```json
{ "ok": true }
```

**エラー時**
```json
{ "error": "エラーメッセージ" }
```

**ステータスコード**

| コード | 意味 |
|--------|------|
| 200 | 成功 |
| 401 | 認証エラー（トークン未指定・無効） |
| 403 | 権限エラー（管理者権限が必要） |
| 409 | 競合エラー（IDが既に使われている） |
| 503 | サービス利用不可（KV接続エラー） |

## 9. 制限事項・注意点

| 項目 | 内容 |
|------|------|
| パスワード管理 | 現バージョンは平文保存。**社外公開・本番運用前にハッシュ化対応が必要** |
| データ容量 | Vercel KV の無料プランは1リクエスト最大20MBまで（`express.json` の制限と同様） |
| セッション有効期限 | 30日。期限切れ後は再ログインが必要 |
| Gmail API | ブラウザのポップアップブロックが有効だとOAuth認証ウィンドウが開かない場合あり |
| Gemini API | 個人用APIキーが未設定の場合、共通キーを使用。APIキーの利用上限に注意 |
| カレンダー検索 | Google Calendar APIの無料枠（1日1億クエリ）内での利用を想定 |
| マルチテナント | 現バージョンは1デプロイ＝1チームの構成。複数チームへの提供は別デプロイが必要 |

## 10. 初回利用の流れ

```
1. Vercelにデプロイする
2. ブラウザでアプリにアクセスする
3. 「サインアップ」から管理者アカウントを作成する
4. ログインする
5. 設定画面でGemini APIキーを入力する（AIアシスタントを使う場合）
6. 設定画面でGmail Client IDを入力する（Gmail連携を使う場合）
7. マスター設定でメンバー・ステータス・流入元を登録する
8. アカウント管理でIS担当者のアカウントを追加する
9. リード管理から顧客データを登録し始める
```
